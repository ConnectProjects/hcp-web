1. INTRODUCTION

What is MasterDB & TechTool?

The Connect Hearing Industrial Division's HCP platform is a professional audiometric management system consisting of two specialized components:

MasterDB: The central database used at the office. It manages all historical employee records, company visit schedules, and generates testing instructions (packets).

TechTool: The field testing application. Technicians use this on-site to record audiometer thresholds, automatically classify results, and counsel workers. TechTool works entirely offline once synchronized.

Privacy & Security: All patient data resides within the browser's secure sandboxed storage (OPFS for MasterDB, IndexedDB for TechTool). No clinical data is stored on external cloud servers owned by the developer. Access is governed by your organization's OneDrive permissions.

2. MASTERDB: OFFICE ADMINISTRATION

Dashboard & Monitoring

The Dashboard is the primary monitoring console for the hearing conservation program. It displays critical KPI tiles:

  • Companies: total active client companies.
  • Active Employees: total individual workers eligible for testing.
  • Tests (30 days): current testing volume.
  • Incoming Packets: completed visits waiting to be imported from technicians.
  • Pending (in field): packets assigned but not yet returned.

Company Management

Companies are the root of all data in MasterDB. Each company record includes:

  • Province Selection: Determines the regulatory rule set (AB, BC, or SK) for all tests at that site.
  • Sticky Notes: Special instructions (access codes, booth location) that appear on the technician's screen during the visit.

Employee Records

Every employee must have a profile before being included in a test packet. Employees should have an Active Baseline — the reference audiogram for all future periodic shift calculations. Without a baseline, periodic shifts (STS) cannot be determined.

Manual Test Entry: Admins can manually enter test results for data correction or historical paper records via the Employee Detail screen.

Packet Management

A packet is a bundle of data for an on-site visit. The lifecycle is:

  1. Generate: Admins select a company, technician, and employee list.
  2. Export: The file is written to the technician's unique folder in the OneDrive Sync Folder.
  3. Submit: The technician drops the completed file into the "Inbox."
  4. Import/Reject: Admins review results and either write them to the permanent database or reject the packet if errors are found.

Legacy Excel Importer

Used to migrate historical data from old Excel workbooks (2009-2025):

  • Auto-Header Mapping: Recognizes columns like "Left 05 KHZ" regardless of position.
  • Date Parsing: Handles variations like "JUL", "SEPT", and full names.
  • Conflict Resolution: If a name in Excel matches an entry in MasterDB but DOBs differ, the admin must manually confirm or skip the match.

3. TECHTOOL: FIELD OPERATIONS

Initial Setup & Sync

Technicians must configure their profile with their name, initials, and folder path. To work on-site:

  1. Open TechTool while connected to the internet.
  2. Tap Sync to download assigned packets.
  3. Once downloaded, all testing, classification, and counseling can be done offline.

The Testing Workflow

For every worker in the visit:

  1. Entry: Select thresholds from the dropdowns (0 dB to 100 dB, or NR).
  2. Classification: TechTool automatically applies the provincial rules.
  3. Counsel: TechTool generates a pre-formatted text based on the category.
  4. Finalize: Confirm the result and save to the local field packet.

British Columbia (WorkSafeBC)

  • Baseline: N (Normal), EW (Early Warning - 15dB notch), A (Abnormal - 30dB@low freq).
  • Periodic: NC (Normal Change), EWC (shift ≥ 15dB at 3k/4k), AC (shift ≥ 15dB at 2 adjacent freq).

Saskatchewan

  • N: Normal.
  • EW: Shift ≥ 15 dB at single freq 2k-6k.
  • A: Avg threshold ≥ 25 dB (500-6k); or shift ≥ 15 dB at 2 adjacent; or shift ≥ 25 dB at any single frequency.

Referrals & Counselling

Whenever an Abnormal result is triggered, a medical referral is required. Technicians should:

  1. Complete the paper referral forms on-site.
  2. (Optional) Print the Referral Form directly from TechTool if a printer is available.

4. DATA INTEGRITY & BACKUPS

Administrative Safety Protocols

  • Excel Redundancy: MasterDB automatically exports all tables to the `excel/` folder in the Sync directory as a readable "shadow" record of the database.
  • Auto-Backups: The system regularly saves `.sqlite` snapshots to the `backups/` folder in OneDrive.
  • Critical Manual Backup: Admins MUST use the Download Backup button in MasterDB Settings after every import and store it securely on a server.
