1. Platform Overview

The HCP platform uses a two-app workflow to manage industrial audiometric testing:

  • MasterDB: The central repository at the office. Used for managing historical data, generating testing "packets," and importing field results.
  • TechTool: The field application used on-site. Works fully offline. Techs download packets, conduct tests, and submit completed results back to the office.

Communication happens via a shared Sync Folder (typically OneDrive), allowing for seamless data exchange without direct database access.

2. MasterDB: Office Administration

Dashboard & KPIs

The Dashboard provides an immediate health check of the program:

  • Companies: Total active companies.
  • Active Employees: Total employees currently in the program.
  • Tests (30 days): Recent testing volume.
  • Incoming Packets: Number of completed visits waiting to be imported.
  • Pending (in field): Packets currently assigned to technicians but not yet returned.

Company & Employee Management

MasterDB allows full CRUD (Create, Read, Update, Delete) operations:

  • Companies: Register new companies, set provincial rules, and maintain "Sticky Notes" for field technicians.
  • Employees: Maintain employee profiles including Hire Date, Date of Birth, and Job Titles.
  • Baselines: Set and manage the reference audiogram for every worker.

Packet Workflow

The core of the system is the Packet:

  1. Generate: Select a company, a technician, and the employees to be tested.
  2. Export: MasterDB saves a JSON packet to the technician's subfolder in the Sync Folder.
  3. Review & Import: Once submitted, MasterDB scans the "Inbox," provides a review of results, and imports them.
  4. Rejection: If a packet is incorrect, use the "Reject" button to clear it from the queue.

3. TechTool: Field Testing

Getting Started & Syncing

TechTool is designed for offline use in the field:

  • Login: Techs log in with assigned initials and folder name.
  • Download Sync: tap "Sync" while connected to download assigned visits.
  • Offline Operation: Once synced, the application requires no internet to test or classify results.

Testing Workflow

  1. Pre-Test Questionnaire: Record medical history and noise habits using a clear, card-based layout.
  2. Threshold Entry: Enter thresholds from the audiometer (0.5k to 8k Hz).
  3. Classification: The system automatically applies provincial rules (Alberta, BC, etc.) to determine if the result is Normal, Early Warning, or Abnormal.
  4. Counsel & Finalize: Refine the auto-generated counsel text and save the test locally.

4. Reports & Data Export

Beyond standard regulatory reports, the platform provides flexible data portability:

  • Excel/CSV Export: MasterDB automatically exports key database tables to an "excel/" subfolder in your Sync Folder. Administrative staff can open these files in Excel or Power BI for analysis without needing MasterDB access.
  • Regulatory Reporting: Generate Company Annual Reports and Employee History Reports as high-quality PDFs.

5. Privacy & Security

The HCP platform protects sensitive worker health information through:

  • Local-First Storage: All clinical data is stored in the local browser's Origin Private File System. Data is never sent to a central developer cloud.
  • Access Control: Access to packets and backups is governed by your organization's existing OneDrive/SharePoint permissions.
  • Sandboxed Environment: Applications run in secure browser sandboxes to prevent unauthorized system access.

6. Data Integrity & Backups

Automated Backups

MasterDB automatically attempts to save a full database snapshot (.sqlite format) to a "backups/" folder in the Sync Folder regularly.

Critical Admin Task: Because data is stored in the browser, clearing your computer's browser history could delete the database. Always use the "Download Backup" button in Settings to save an external copy after every major import session.
