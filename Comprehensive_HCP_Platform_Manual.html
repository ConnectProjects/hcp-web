I. System Introduction & Workflow

The Connect Hearing Industrial Division's Hearing Conservation Platform (HCP) is a specialized suite of tools designed to manage large-scale industrial audiometric programs. The system separates high-security office administration from high-efficiency field testing.

The Two-App Architecture

  1. MasterDB: The "Brain" of the operation. Used by administrative staff to manage the database, generate testing packets, and import results. It runs in a browser and stores data locally via the Origin Private File System (OPFS).
  2. TechTool: The "Hands" of the operation. Used by technicians on mobile devices/laptops at client sites. It works entirely offline and uses IndexedDB for local storage.
The Sync Folder Mechanism: Communication between apps occurs via a shared folder (e.g., OneDrive). Files are swapped between an Outbox (packets sent to techs) and an Inbox (completed results sent to the office).

Company & Employee Management

MasterDB handles the complete management of industrial assets:

  • Regulatory Provisioning: Each company is assigned a province (AB, BC, SK), which determines the classification engine used for their tests.
  • Sticky Notes: Admins can leave "Site Instructions" that appear directly on the technician's device during testing.
  • Employee Records: Maintains complete history, including Hire Date and Date of Birth. DOB is used for robust matching during complex data imports.
  • Manual Entries: Admins can manually enter or edit test results (via the "Manual Test Entry" button) to correct errors or input historical paper records.

Packet Lifecycle

Testing "Packets" govern the movement of data:

1
Generate: Select employees and assign a technician.
2
Pending: The packet awaits pickup in the technician's sync folder.
3
Submitted: The technician has finished and sent it back to the office inbox.
4
Imported/Rejected: The office reviews the results and either writes them to the database or rejects the packet for re-processing.

Legacy Data Migration (Excel Import)

MasterDB includes a powerful multi-header Excel parser for historical data:

  • Scans spreadsheets to map columns like "Left 05 KHZ" or "Test Date" automatically.
  • Handles date variations (e.g., "SEPT", "SEP", "SEPTEMBER").
  • Includes a conflict resolution screen for duplicate names to ensure integrity during migration.

III. TechTool: Field Operations

Field Synchronization & Offline Setup

Technicians begin their day with a "Sync" action:

  1. Open TechTool while connected to the internet.
  2. Tap Sync to download assigned visits.
  3. Disconnect and head to the site. The application is now fully self-contained.

Conducting Tests: Entry & Classification

The testing interface is optimized for speed and accuracy:

  • Threshold Dropdowns: Only allow valid 5 dB steps from 0 to 100 dB.
  • NR (No Response) Support: Handles extreme hearing loss for specialist referral.
  • Real-time Audiogram: Updates a 7-frequency chart visually as the tech types.
  • Classification: Automatically assigns a category based on the current thresholds compared to the worker's baseline.

Counselling & Referrals

TechTool generates automated counselling templates. For Abnormal or STS outcomes:

  • The system alerts the technician that a referral is required.
  • A Referral Form can be generated and printed/saved on-site.
  • The tech must confirm they have provided the worker with the necessary info before finalizing.

IV. Technical Appendices

Provincial Rule Reference

Below are the core classification logic points used by the system:

Alberta (OHS Code Part 16)

OutcomeCodeTrigger Condition
NormalNNo rules triggered.
Standard Threshold ShiftEWAvg shift ≥ 10 dB at 2K, 3K, 4K Hz vs baseline.
AbnormalAThreshold > 25 dB at 500-2000Hz OR asymmetric > 30 dB.

British Columbia (WorkSafeBC)

  • Baseline Tests: Produce N (Normal), EW (Early Warning), or A (Abnormal).
  • Periodic Tests: Produce NC (Normal Change), EWC (Early Warning Change), or AC (Abnormal Change).
  • EWC Trigger: Single frequency shift ≥ 15 dB at 3K or 4K vs baseline.

Security, Privacy & Backup Protocols

Privacy Directive: All PII (Personally Identifiable Information) and test history reside within the browser's Origin Private File System. MasterDB does NOT use an external API for data storage.

Backups & Data Portability

  • Manual Database Backup: Administrators must download the .sqlite file via Settings after major imports. This is the primary recovery point.
  • Excel Shadow Backup: MasterDB automatically exports CSV copies of all tables to the excel/ folder in the Sync directory to ensure data is readable even without the app.
  • Packet Pruning: TechTool automatically clears temporary field packets once submission is verified to maintain local device privacy.
